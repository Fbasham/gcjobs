Frequently Asked Questions

On the home page, enter your query into the search bar. Try using single words to get the widest search possible. If you'd like to refine your search using multiple criteria, use comma separated values. For example,  python, engineering, toronto, AS-05  would return all posters that contain "python" and "engineering" and "toronto" and "AS-05". The search is case insensitive.
The most likely reason is that the script to pull new jobs into the database for this site is only run once or twice per day. Depending on the time you search, the results can differ. I do plan on experimenting with the frequency of running the script, but on a day-by-day basis once or twice per day seems reasonable.

The results you see on the home page are all of the job posters currently available at the last time they were pulled into the database. If a poster expires and is no longer viewable on the office GC Jobs website, it will eventually be dropped on here as well.
Currently AND is the only logical operator being supported and uses the comma separated syntax outlined under "How do I use the search bar?".
Depending on the search criteria, some results could be unexpected. For example, typing  javascript  into the search bar will likely return some unrelated job posters. This is because "javascript" could be mentioned in the HTML content of the page, specifically in a script tag, despite not having anything to do with the actual job itself. For most searches, this will be irrelevant, but keep in mind that some searches might need to be refined.
