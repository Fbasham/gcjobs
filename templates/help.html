Frequently Asked Questions

On the home page, enter your query into the search bar. Try using single words to get the widest search possible. If you'd like to refine your search using multiple criteria, use comma separated values. For example, python,engineering,toronto,AS-05 would return all posters that contain "python" and "engineering" and "toronto" and "AS-05". The search is case insensitive.

I've created a subreddit because it's widely used and there is an anonymous component if you'd like to remain hidden behind a throwaway account. Another option could be to add some additional support on this website in a future update.
The results you see on the home page are all of the external job posters currently available at the last time they were pulled into the database. If a poster expires and is no longer viewable on the office GC Jobs website, it will eventually be dropped on here as well.
Currently AND is the only logical operator being supported and uses the comma separated syntax outlined under "How do I use the search bar?" In a future update I might include additional syntax for OR, NOT, and a combination or AND, OR, and NOT. I don't envision there being a pressing need for such searches though, which is why by default AND is the only operator supported. The same goes for regex searches.
Depending on the search criteria, some results could be unexpected. For example, typing javascript into the search bar will likely return some unrelated job posters. This is because "javascript" could be mentioned in the HTML content of the poster despite not having anything to do with the actual job itself. For most searches, this will be irrelevant, but keep in mind that some searches might need to be refined.
