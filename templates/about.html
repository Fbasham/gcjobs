About

The intent of this project is extend the functionality of the official GC Jobs website. Currently, there's no way to search posters for keywords. Thus, the user is left to helplessly guess, or worse, individually open job posters to find the content they might be looking for. When it comes to job hunting, why spend the bulk of your time searching for the perfect oppourtunity? Save it for the screening questions 😉


How it Works

This site is written in pure Python. The official GC Jobs website only lets you do a partial text search by job title or location. Often times the job title isn't 100% reflective of the actual duties you might expect to carry out day-to-day.

Unfortunately there isn't a public API to hook into to conveniently grab job content in JSON format. Another shortcoming is that the GC Jobs website is heavily rendered in Javascript to initially load and paginate all of the job posters. This makes it slightly more difficult to cleanly grab content as quickly as possible using Python. The workaround for this is to use Selenium, a browser automation/simulating tool, to navigate through through the GC Jobs website, collect the links to each poster, and store them in a list.

Once all of the links have been collected, asynchronous HTTP requests can be made to each poster since the content isn't actually rendered in Javascript. The HTML content of the posters and other pertinent fields are then written to a database where it can be efficiently queried using an object-relational mapper (ORM). In case you're wondering what third party packages I used for this project: 

 apscheduler        (scheduling background tasks)
 beautifulsoup4     (HTML parsing)
 Flask              (web framework)
 gunicorn           (Python WSGI HTTP Server for UNIX)
 psycopg2           (Postgres database adapter)
 requests-html      (HTTP requests/HTML parsing)
 selenium           (browser automation for Javascript rendering)
 SQLAlchemy         (ORM for database)

The list of standard library modules is even smaller: 

 asyncio            (asynchronous utilities)
 re                 (regular expressions)
